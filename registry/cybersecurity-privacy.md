# Cybersecurity and Privacy

Information security management, cybersecurity frameworks, data protection
regulations, and privacy standards. Covers the ISO 27000 series, Common
Criteria, industrial control system security (IEC 62443), NIST publications,
threat intelligence databases, and global data protection laws (GDPR, CCPA,
HIPAA, LGPD, and others).

**Contents:** [Standard](#standard) | [Regulation](#regulation) |
[Publication](#publication)

## Standard

- [IEC-62443] Industrial communication networks — IT security for networks and
  systems

  Comprehensive series for industrial automation and control system (IACS)
  cybersecurity. Defines security levels (SL 1-4), zones and conduits,
  requirements for operators, integrators, and component suppliers.

  Document: IEC 62443 (series)\
  URL: <https://webstore.iec.ch/en/publication/33615>\
  Label: Standard\
  Keywords: industrial, OT security, IACS

- [ISO-15408] Information technology — Security techniques — Evaluation criteria
  for IT security

  Common Criteria (CC) for information technology security evaluation. Defines
  Evaluation Assurance Levels (EAL 1-7) for certifying the security properties
  of IT products and systems.

  Document: ISO/IEC 15408:2022 (all parts)\
  URL: <https://www.iso.org/standard/72891.html>\
  Label: Standard\
  Keywords: common criteria, evaluation, assurance

- [ISO-27001] Information security, cybersecurity and privacy protection —
  Information security management systems

  Requirements for establishing, implementing, maintaining, and continually
  improving an information security management system (ISMS). The globally
  recognized certification standard for information security.

  Document: ISO/IEC 27001:2022\
  URL: <https://www.iso.org/standard/27001>\
  Label: Standard\
  Keywords: information security, ISMS, certification

- [ISO-27002] Information security, cybersecurity and privacy protection —
  Information security controls

  Reference set of information security controls. Provides implementation
  guidance for the controls referenced in ISO 27001 Annex A. Organized by
  themes: organizational, people, physical, and technological.

  Document: ISO/IEC 27002:2022\
  URL: <https://www.iso.org/standard/75652.html>\
  Label: Standard\
  Keywords: information security, controls\
  Derived-from: ISO-27001

- [ISO-27005] Information security, cybersecurity and privacy protection —
  Guidance on managing information security risks

  Guidelines for information security risk management. Supports the risk
  assessment and treatment requirements defined in ISO 27001. Provides
  systematic approaches for identifying, analyzing, and evaluating risks.

  Document: ISO/IEC 27005:2022\
  URL: <https://www.iso.org/standard/80585.html>\
  Label: Standard\
  Keywords: information security, risk management\
  Derived-from: ISO-27001

- [ISO-27017] Information technology — Security techniques — Code of practice
  for information security controls based on ISO/IEC 27002 for cloud services

  Cloud-specific security guidance extending ISO 27002 controls. Addresses
  shared responsibility between cloud service providers and customers, virtual
  machine hardening, and cloud-specific operational procedures.

  Document: ISO/IEC 27017:2015\
  URL: <https://www.iso.org/standard/43757.html>\
  Label: Standard\
  Keywords: cloud security, information security\
  Derived-from: ISO-27002

- [ISO-27018] Information technology — Code of practice for protection of
  personally identifiable information (PII) in public clouds

  Code of practice for protecting PII processed by public cloud service
  providers. Extends ISO 27002 with cloud-specific PII protection controls
  covering consent, data minimization, and cross-border transfers.

  Document: ISO/IEC 27018:2019\
  URL: <https://www.iso.org/standard/76559.html>\
  Label: Standard\
  Keywords: cloud, PII, privacy\
  Derived-from: ISO-27001

- [ISO-27034] Information technology — Application security

  Framework for application security across the life cycle. Defines an
  Organization Normative Framework (ONF) and Application Normative Framework
  (ANF) for integrating security into software development.

  Document: ISO/IEC 27034 (series)\
  URL: <https://www.iso.org/standard/44378.html>\
  Label: Standard\
  Keywords: application security, software development\
  Derived-from: ISO-27001

- [ISO-27701] Security techniques — Extension to ISO/IEC 27001 and ISO/IEC 27002
  for privacy information management

  Specifies requirements and guidance for establishing a privacy information
  management system (PIMS) as an extension to ISO 27001 and ISO 27002. Maps to
  GDPR requirements and supports compliance demonstration.

  Document: ISO/IEC 27701:2019\
  URL: <https://www.iso.org/standard/71670.html>\
  Label: Standard\
  Keywords: privacy management, PIMS\
  Derived-from: ISO-27001

- [ISO-29100] Information technology — Privacy framework

  Defines a privacy framework including privacy principles, actors, and roles.
  Establishes common terminology and concepts for privacy protection in
  information systems.

  Document: ISO/IEC 29100:2024\
  URL: <https://www.iso.org/standard/85938.html>\
  Label: Standard\
  Keywords: privacy framework, terminology

- [ISO-29134] Information technology — Guidelines for privacy impact assessment

  Guidelines for conducting privacy impact assessments (PIAs). Defines a
  structured process for identifying, assessing, and mitigating privacy risks in
  systems that process personal data.

  Document: ISO/IEC 29134:2023\
  URL: <https://www.iso.org/standard/86012.html>\
  Label: Standard\
  Keywords: privacy impact assessment, PIA

- [PCI-DSS] Payment Card Industry Data Security Standard

  Security standard for organizations that handle payment card data. Defines
  twelve requirement groups covering network security, access control,
  encryption, vulnerability management, and monitoring. Required for merchants,
  processors, and service providers.

  Document: PCI DSS v4.0.1 (2024)\
  URL: <https://www.pcisecuritystandards.org/standards/pci-dss/>\
  Label: Standard\
  Keywords: payment card, financial

- [SOC-2] Service Organization Control 2

  AICPA audit framework for evaluating service organisations' controls relevant
  to security, availability, processing integrity, confidentiality, and privacy.
  SOC 2 Type II reports cover control effectiveness over a period. Required by
  enterprise customers for cloud and SaaS providers.

  Document: AICPA SOC 2 (TSP Section 100)\
  URL:
  <https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2>\
  Label: Standard\
  Keywords: audit, cloud, SaaS

## Regulation

- [APPI] Act on the Protection of Personal Information

  Japan's comprehensive data protection law. Establishes rules for the handling
  of personal information by business operators including consent requirements,
  purpose limitation, and cross-border transfer restrictions. Enforced by the
  Personal Information Protection Commission (PPC).

  Document: Act No. 57 of 2003 (amended 2022)\
  URL: <https://www.ppc.go.jp/en/legal/>\
  Label: Regulation\
  Keywords: personal data, Japan

- [CCPA] California Consumer Privacy Act

  California state law granting consumers rights over personal information
  collected by businesses. Includes rights to know, delete, opt-out, and
  non-discrimination. Amended by the CPRA (2023).

  Document: Cal. Civ. Code 1798.100-1798.199.100\
  URL: <https://oag.ca.gov/privacy/ccpa>\
  Label: Regulation\
  Keywords: consumer privacy, California, US

- [DPDP-Act] Digital Personal Data Protection Act (India)

  India's data protection law establishing obligations for processing digital
  personal data. Covers consent management, data principal rights, data
  fiduciary obligations, and cross-border transfer requirements with significant
  financial penalties.

  Document: Act No. 22 of 2023\
  URL: <https://www.meity.gov.in/data-protection-framework>\
  Label: Regulation\
  Keywords: personal data, India

- [EU-CRA] Cyber Resilience Act

  EU regulation establishing cybersecurity requirements for products with
  digital elements. Mandates security-by-design, vulnerability handling, and
  software bill of materials (SBOM) for hardware and software products placed on
  the EU market.

  Document: Regulation (EU) 2024/2847\
  URL: <https://eur-lex.europa.eu/eli/reg/2024/2847/oj>\
  Label: Regulation\
  Keywords: product security, SBOM, EU

- [EU-DORA] Digital Operational Resilience Act

  EU regulation on digital operational resilience for the financial sector.
  Requires financial entities to manage ICT risk, report incidents, test
  resilience, and oversee third-party ICT service providers.

  Document: Regulation (EU) 2022/2554\
  URL: <https://eur-lex.europa.eu/eli/reg/2022/2554/oj>\
  Label: Regulation\
  Keywords: financial, operational resilience, EU

- [EU-NIS2] Directive on measures for a high common level of cybersecurity
  across the Union

  EU directive establishing cybersecurity risk management and incident reporting
  obligations for essential and important entities across critical sectors.
  Expands the scope of the original NIS Directive and introduces stricter
  enforcement.

  Document: Directive (EU) 2022/2555\
  URL: <https://eur-lex.europa.eu/eli/dir/2022/2555/oj>\
  Label: Regulation\
  Keywords: critical infrastructure, incident reporting, EU

- [GDPR] General Data Protection Regulation

  European Union regulation on data protection and privacy. Establishes rights
  for data subjects, obligations for data controllers and processors, and
  enforcement mechanisms including significant financial penalties.

  Document: Regulation (EU) 2016/679\
  URL: <https://eur-lex.europa.eu/eli/reg/2016/679/oj>\
  Label: Regulation\
  Keywords: data protection, EU, personal data

- [HIPAA] Health Insurance Portability and Accountability Act

  U.S. federal law establishing privacy and security requirements for protected
  health information (PHI). Includes the Privacy Rule, Security Rule, and Breach
  Notification Rule governing healthcare providers, insurers, and their business
  associates.

  Document: Public Law 104-191 (1996)\
  URL: <https://www.hhs.gov/hipaa/index.html>\
  Label: Regulation\
  Keywords: health data, US, PHI

- [LGPD] Lei Geral de Protecao de Dados Pessoais

  Brazil's general data protection law modelled on the GDPR. Establishes
  principles, data subject rights, and obligations for data controllers and
  processors. Enforced by the Autoridade Nacional de Protecao de Dados (ANPD).

  Document: Lei 13.709/2018\
  URL:
  <https://www.planalto.gov.br/ccivil_03/_ato2015-2018/2018/lei/l13709.htm>\
  Label: Regulation\
  Keywords: data protection, Brazil

- [PIPA] Personal Information Protection Act (South Korea)

  South Korea's general data protection law. Establishes strict consent
  requirements, data minimisation principles, and data breach notification
  obligations. Enforced by the Personal Information Protection Commission (PIPC)
  with significant penalties.

  Document: Act No. 10465 (amended 2023)\
  URL: <https://www.pipc.go.kr/eng/index.do>\
  Label: Regulation\
  Keywords: personal data, South Korea

- [PIPEDA] Personal Information Protection and Electronic Documents Act

  Canadian federal privacy law governing how private-sector organizations
  collect, use, and disclose personal information in the course of commercial
  activities. Based on ten fair information principles.

  Document: S.C. 2000, c. 5\
  URL: <https://laws-lois.justice.gc.ca/eng/acts/p-8.6/>\
  Label: Regulation\
  Keywords: personal data, Canada

- [UK-DPA-2018] Data Protection Act 2018

  UK domestic implementation of data protection law post-Brexit. Supplements the
  UK GDPR with provisions for law enforcement processing, intelligence services,
  and specific UK derogations from GDPR requirements.

  Document: Data Protection Act 2018\
  URL: <https://www.legislation.gov.uk/ukpga/2018/12/contents>\
  Label: Regulation\
  Keywords: data protection, UK

## Publication

- [CWE] Common Weakness Enumeration

  Community-developed catalogue of software and hardware weakness types.
  Provides a standardised taxonomy for vulnerabilities used by static analysis
  tools, code review processes, and secure development standards including OWASP
  and CERT.

  Document: CWE 4.16 (2024)\
  URL: <https://cwe.mitre.org/>\
  Label: Publication\
  Keywords: vulnerability, weakness, taxonomy

- [FedRAMP] Federal Risk and Authorization Management Program

  U.S. government programme standardising security assessment and authorisation
  of cloud services used by federal agencies. Defines three impact levels (Low,
  Moderate, High) with controls based on NIST SP 800-53.

  Document: FedRAMP Authorization Framework\
  URL: <https://www.fedramp.gov/>\
  Label: Publication\
  Keywords: cloud, federal, authorization

- [MITRE-ATTCK] MITRE ATT&CK — Adversarial Tactics, Techniques, and Common
  Knowledge

  Knowledge base of adversary tactics and techniques based on real-world
  observations. Covers Enterprise, Mobile, and ICS matrices. Used for threat
  modelling, detection engineering, red teaming, and security gap analysis.

  Document: ATT&CK v16 (2024)\
  URL: <https://attack.mitre.org/>\
  Label: Publication\
  Keywords: threat intelligence, tactics, techniques

- [NIST-CSF] NIST Cybersecurity Framework

  Voluntary framework for managing cybersecurity risk. Organized around six
  functions: Govern, Identify, Protect, Detect, Respond, Recover. Widely adopted
  across industries and referenced by regulators.

  Document: NIST CSF 2.0 (2024)\
  URL: <https://www.nist.gov/cyberframework>\
  Label: Publication\
  Keywords: risk management, framework

- [NIST-SP-800-171] Protecting Controlled Unclassified Information in Nonfederal
  Systems and Organizations

  Security requirements for protecting controlled unclassified information (CUI)
  in non-federal systems. Required for U.S. defense contractors and widely used
  as a baseline for supply chain security assurance.

  Document: NIST SP 800-171 Rev. 3 (2024)\
  URL: <https://csrc.nist.gov/pubs/sp/800/171/r3/final>\
  Label: Publication\
  Keywords: CUI, defense, supply chain

- [NIST-SP-800-53] Security and Privacy Controls for Information Systems and
  Organizations

  Comprehensive catalogue of security and privacy controls for federal
  information systems. Defines control families covering access control, audit,
  incident response, system integrity, and more. Widely adopted beyond
  government as a baseline for enterprise security.

  Document: NIST SP 800-53 Rev. 5 (2020)\
  URL: <https://csrc.nist.gov/pubs/sp/800/53/r5/upd1/final>\
  Label: Publication\
  Keywords: security controls, federal

- [NIST-SP-800-82] Guide to Operational Technology (OT) Security

  Guidance for securing operational technology in industrial control systems,
  SCADA, DCS, and other OT environments. Addresses threats specific to
  industrial environments and provides security architecture recommendations for
  convergent IT/OT networks.

  Document: NIST SP 800-82 Rev. 3 (2023)\
  URL: <https://csrc.nist.gov/pubs/sp/800/82/r3/final>\
  Label: Publication\
  Keywords: OT security, industrial, SCADA

- [OWASP-ASVS] Application Security Verification Standard

  Framework of security requirements and controls for web application security.
  Defines three verification levels with requirements covering authentication,
  session management, access control, input validation, and cryptography.

  Document: OWASP ASVS 4.0.3 (2021)\
  URL:
  <https://owasp.org/www-project-application-security-verification-standard/>\
  Label: Publication\
  Keywords: application security, web, verification
