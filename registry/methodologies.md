# Methodologies and Frameworks

## Agile

- [Agile-Manifesto] Manifesto for Agile Software Development

  Foundational values and twelve principles for iterative, collaborative
  software development. Prioritises individuals and interactions, working
  software, customer collaboration, and responding to change over rigid
  processes and documentation.

  Document: Agile Manifesto (2001)\
  URL: <https://agilemanifesto.org/>

- [Scrum-Guide] The Scrum Guide

  Defines the Scrum framework for developing, delivering, and sustaining complex
  products. Specifies roles (Product Owner, Scrum Master, Developers), events
  (Sprint, Daily Scrum, Review, Retrospective), and artifacts (Product Backlog,
  Sprint Backlog, Increment).

  Document: The Scrum Guide (2020)\
  URL: <https://scrumguides.org/>

- [SAFe] Scaled Agile Framework

  Framework for scaling agile practices across large enterprises. Defines four
  configurations (Essential, Large Solution, Portfolio, Full) with Agile Release
  Trains (ARTs), PI Planning, and lean portfolio management. Widely adopted in
  automotive, aerospace, and defence software.

  Document: SAFe 6.0 (2023)\
  URL: <https://scaledagileframework.com/>

- [Kanban] Kanban Method

  Lean method for managing knowledge work using visual boards, WIP limits, and
  flow metrics. Provides an evolutionary approach to process improvement without
  prescribed roles or ceremonies. Based on Toyota Production System principles.

  Document: Kanban Guide (2020)\
  URL: <https://kanbanguides.org/>

## Systems Development

- [V-Model] V-Model Systems Development Process

  Sequential development model mapping each design phase to a corresponding test
  phase in a V-shape. Mandated by IEC 61508, ISO 26262, EN 50128, and DO-178C as
  the reference lifecycle for safety-critical systems development.

  Document: VDI 2206:2021 / V-Modell XT 2.3\
  URL:
  <https://www.vdi.de/richtlinien/details/vdi-2206-development-of-mechatronic-and-cyber-physical-systems>

- [PRINCE2] Projects IN Controlled Environments

  Structured project management method defining seven principles, themes, and
  processes. Covers business justification, organisation, quality, risk, change,
  plans, and progress. Widely adopted in UK government and European projects.

  Document: PRINCE2 7th Edition (2023)\
  URL: <https://www.axelos.com/certifications/prince2>

## IT Service Management

- [ITIL] Information Technology Infrastructure Library

  IT service management (ITSM) framework defining practices for aligning IT
  services with business needs. Covers service strategy, design, transition,
  operation, and continual improvement. The globally dominant ITSM reference
  framework.

  Document: ITIL 4 (2019)\
  URL: <https://www.axelos.com/certifications/itil-service-management>

## DevOps and Continuous Delivery

- [DORA-Metrics] DORA DevOps Metrics

  Four key metrics for measuring software delivery and operational performance:
  deployment frequency, lead time for changes, change failure rate, and time to
  restore service. Based on the Accelerate / State of DevOps research programme.

  Document: Accelerate (2018) / DORA (Google Cloud)\
  URL: <https://dora.dev/>

- [Twelve-Factor] The Twelve-Factor App

  Methodology for building software-as-a-service applications. Defines twelve
  principles covering codebase, dependencies, config, backing services,
  build/release/run, processes, port binding, concurrency, disposability,
  dev/prod parity, logs, and admin processes.

  Document: The Twelve-Factor App (2017)\
  URL: <https://12factor.net/>

## Process Improvement

- [Lean] Lean Manufacturing / Lean Thinking

  Management philosophy derived from the Toyota Production System. Focuses on
  eliminating waste (muda), reducing variability (mura), and avoiding overburden
  (muri) through value stream mapping, just-in-time, and continuous improvement
  (kaizen).

  Document: Toyota Production System / Lean Enterprise Institute\
  URL: <https://www.lean.org/>

- [Six-Sigma] Six Sigma

  Data-driven quality methodology using DMAIC (Define, Measure, Analyze,
  Improve, Control) for process improvement and DMADV for new process design.
  Aims for 3.4 defects per million opportunities. Widely combined with Lean as
  Lean Six Sigma.

  Document: ISO 13053:2011\
  URL: <https://www.iso.org/standard/52901.html>

## Design

- [Design-Thinking] Design Thinking

  Human-centred innovation methodology using five phases: Empathize, Define,
  Ideate, Prototype, Test. Encourages rapid prototyping and iterative learning.
  Applied across product design, service design, and organisational change.

  Document: ISO 56002:2019 (Innovation management)\
  URL: <https://www.iso.org/standard/68221.html>
