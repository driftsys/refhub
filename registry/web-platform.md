# Web Platform

## Core Standards

- [HTML] HyperText Markup Language

  The standard markup language for documents displayed in web browsers. Defines
  the structure and semantics of web content including forms, multimedia
  embedding, accessibility attributes, and document metadata. Maintained as a
  living standard by WHATWG.

  Document: HTML Living Standard (WHATWG)\
  URL: <https://html.spec.whatwg.org/>

- [CSS] Cascading Style Sheets

  Style sheet language for describing the visual presentation of HTML documents.
  Covers layout (Flexbox, Grid), typography, colour, animation, responsive
  design, and custom properties. Developed as modular specifications by W3C.

  Document: CSS Snapshot (W3C, annual)\
  URL: <https://www.w3.org/Style/CSS/>

- [ECMAScript] ECMAScript Language Specification

  The standardised specification underlying JavaScript. Defines language syntax,
  types, built-in objects, modules, async/await, and iterators. Annual releases
  add new features. The universal programming language of the web platform.

  Document: ECMA-262 (15th Edition, 2024)\
  URL:
  <https://ecma-international.org/publications-and-standards/standards/ecma-262/>

- [DOM] Document Object Model

  W3C/WHATWG API for programmatically accessing and manipulating HTML and XML
  documents. Defines the tree structure, node types, events, mutation observers,
  and traversal methods used by JavaScript in all browsers.

  Document: DOM Living Standard (WHATWG)\
  URL: <https://dom.spec.whatwg.org/>

- [WebAssembly] WebAssembly

  W3C standard for a portable binary instruction format for a stack-based
  virtual machine. Enables near-native performance in web browsers for
  computationally intensive applications including games, codecs, CAD, and
  scientific simulations.

  Document: WebAssembly Core Specification 2.0 (2024)\
  URL: <https://webassembly.github.io/spec/core/>

- [SVG] Scalable Vector Graphics

  W3C XML-based markup language for two-dimensional vector graphics. Supports
  interactive graphics, animation, and scripting. Used for icons, charts, maps,
  and resolution-independent graphics on the web.

  Document: SVG 2 (W3C, 2018)\
  URL: <https://www.w3.org/TR/SVG2/>

## Data and Serialisation

- [XML] Extensible Markup Language

  W3C standard for structured data representation. Foundation for XSLT, XPath,
  XML Schema, SOAP, and numerous industry data exchange formats (ODX, AUTOSAR
  ARXML, HL7 CDA). Remains dominant in enterprise and safety-critical data
  interchange.

  Document: XML 1.0 (Fifth Edition, 2008)\
  URL: <https://www.w3.org/TR/xml/>

- [Unicode] The Unicode Standard

  Universal character encoding standard assigning unique code points to every
  character in every writing system. Covers encoding forms (UTF-8, UTF-16,
  UTF-32), normalisation, bidirectional text, emoji, and collation. The
  mandatory character set for modern software.

  Document: The Unicode Standard 16.0 (ISO/IEC 10646:2023)\
  URL: <https://www.unicode.org/versions/latest/>

- [ISO-8601] Date and time — Representations for information interchange

  Defines unambiguous date and time formats for data exchange. Specifies
  YYYY-MM-DD dates, 24-hour times, time zones (UTC offsets), durations, and
  intervals. The universal interchange format used in APIs, databases, and
  logging.

  Document: ISO 8601-1:2019\
  URL: <https://www.iso.org/standard/70907.html>

## Web APIs

- [Fetch] Fetch Standard

  WHATWG specification defining the Fetch API for making HTTP requests from
  JavaScript. Covers request/response objects, CORS, streaming bodies, caching,
  and redirects. Replaces XMLHttpRequest as the modern network request API.

  Document: Fetch Living Standard (WHATWG)\
  URL: <https://fetch.spec.whatwg.org/>

- [Web-Components] Web Components

  Collection of W3C standards (Custom Elements, Shadow DOM, HTML Templates)
  enabling reusable, encapsulated UI components in web applications. Allows
  framework-independent component development with native browser support.

  Document: W3C Web Components (2023)\
  URL: <https://www.w3.org/wiki/WebComponents/>

- [WebGPU] WebGPU

  W3C API for high-performance 3D graphics and general-purpose GPU compute in
  web browsers. Successor to WebGL providing modern GPU access aligned with
  Vulkan, Metal, and Direct3D 12 capabilities.

  Document: WebGPU (W3C, 2024)\
  URL: <https://www.w3.org/TR/webgpu/>

- [PWA] Progressive Web Applications

  W3C collection of standards (Web App Manifest, Service Workers, Push API)
  enabling web applications to function like native apps with offline support,
  push notifications, home screen installation, and background sync.

  Document: Web App Manifest (W3C, 2023)\
  URL: <https://www.w3.org/TR/appmanifest/>
